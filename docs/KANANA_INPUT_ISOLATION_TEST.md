# KANANA_INPUT_ISOLATION_TEST

## 왜 분리 테스트가 필요한가

현재 이슈는 모델이 입력(이미지/음성)을 충분히 반영하지 않고 기본 페르소나 응답으로 흐르는 점이다.
입력 채널을 분리하면 원인을 더 빨리 좁힐 수 있다.

- image_only: 이미지 이해가 되는지 확인
- audio_only: 음성 인식/반영이 되는지 확인
- image_audio: 멀티모달 결합 응답 품질 확인

## 현재 관측 결과 (업데이트)

- image_only: **이미지 이해 성공에 가까움**
- audio_only: **핵심 의미 반영은 성공**, 다만 **정확 전사보다 요약 경향**이 있음
- 다음 핵심 테스트: **image_audio 모드에서 이미지+음성 동시 반영 품질 검증**

## 모드별 확인 포인트

### 1) image_only
- 전송: text + image
- 확인: 이미지의 대상/상황을 반영한 답변인지
- 현재 관측: 이미지 이해 성공으로 보임

### 2) audio_only
- 전송: text + audio(wav)
- 확인: 사용자의 마지막 말을 가능한 원문에 가깝게 재현하는지
- 현재 관측: 핵심 의미는 반영하지만, 정확 전사 대신 요약 경향 존재

### 3) image_audio
- 전송: text + image + audio(wav)
- 확인: 그림 속 친구 톤 + 음성 내용 반응이 동시에 나타나는지
- 현재 우선순위: 다음 핵심 검증 대상

## 결과 해석 가이드

- image_only 성공 + audio_only 요약 경향:
  - STT는 동작하나, 전사 정밀도/프롬프트 강도 추가 보정 필요
- image_only 성공 + audio_only 성공 + image_audio 실패:
  - 멀티모달 결합 단계(우선순위/프롬프트) 문제 가능성
- image_audio 성공:
  - 실제 사용자 시나리오(그림+말)에 근접한 품질 확보
