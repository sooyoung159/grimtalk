# CONTINUITY_V2_PLAN

## 왜 지금 구조에서 대화가 끊겨 보이는지
- 첫 턴 이후에도 사실상 다시 생성에 가까운 요청 흐름이라, 같은 친구를 계속 만난다는 감각이 약했다.
- follow-up에서 캐릭터 고정 프로필이 없어서, 응답은 이어져도 정체성과 말투 continuity가 쉽게 흔들렸다.
- 결과 화면에서 다시 말 걸기를 눌러도 사용자는 새 턴을 시작하는 느낌을 받아, 같은 장면 안에서 대화가 이어지는 감각이 약했다.

## 왜 1턴과 후속 턴을 분리해야 하는지
- 첫 턴은 그림 친구가 처음 깨어나는 장면이므로 image + audio가 필요하다.
- 하지만 후속 턴은 이미 만난 같은 친구와 이어서 말하는 구간이므로, 다시 이미지를 보내며 캐릭터를 재생성할 필요가 없다.
- 첫 턴과 후속 턴의 목적이 다르므로 요청 구조도 분리해야 한다.

## 왜 follow-up에서는 image를 다시 보내지 않는지
- 같은 그림을 매 턴 다시 보내면 latency가 커지고, 모델이 같은 친구를 이어서 말하기보다 새로 해석하려는 쪽으로 흐를 수 있다.
- follow-up에서는 이미지 재전송보다 고정 캐릭터 프로필 + 최근 1턴 문맥이 continuity에 더 직접적으로 기여한다.
- audio only로 줄이면 응답 시간이 짧아지고 대화 리듬이 더 살아난다.

## 캐릭터 프로필 고정 전략
- 첫 턴 성공 시 아래 정보를 store에 고정 저장한다.
  - name
  - identity
  - traits
  - voiceTone
  - greeting
  - question
- 후속 턴에서는 이 프로필을 system/user 문맥에 함께 넣어 같은 친구의 말투와 성격을 유지하게 한다.
- 후속 턴 응답이 와도 캐릭터는 새로 만들지 않고 기존 프로필을 유지한다.

## UX continuity 개선 전략
- 다시 말 걸기는 항상 같은 친구에게 이어서 말하는 행동으로 취급한다.
- 후속 턴에서는 audio only로 요청해 처리 시간을 줄인다.
- 최근 대화가 결과 화면에 남아 있는 상태에서 다시 말 걸기를 시작하게 해 장면 continuity를 유지한다.
- 최소 기준으로는 사용자가 방금 한 말과 친구의 직전 말이 남아 있도록 해, 완전히 새 화면으로 끊기는 느낌을 줄인다.

## 기대 효과
- 같은 친구 느낌
  - 캐릭터 정체성/말투/톤이 더 안정적으로 유지된다.
- 응답 속도 개선
  - follow-up에서 image 재전송을 제거해 payload와 처리 비용을 줄인다.
- 아이 몰입도 개선
  - 결과를 다시 만드는 앱이 아니라, 같은 그림 친구와 계속 대화하는 앱에 더 가까워진다.
