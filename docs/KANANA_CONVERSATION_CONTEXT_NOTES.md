# KANANA_CONVERSATION_CONTEXT_NOTES

## 기존 방식(baseText 저장)의 한계

기존 recentUserText는 실제 사용자 발화가 아니라 요청 지시문(baseText)이었다.
예: "이미지 속 친구가 되어..." 같은 문장은 사용자가 실제로 말한 내용이 아님.

문제점:
- 다음 턴에서 모델이 사용자 의도를 연속적으로 추적하기 어려움
- 대화 맥락이 "지시문 반복"으로 보이고, 자연스러운 이어말하기 품질이 떨어짐

## 왜 실제 사용자 발화 기반 recentUserText가 중요한가

대화 품질은 "직전 사용자가 무엇을 말했는지"를 정확히 기억하는 데 크게 좌우된다.
recentUserText가 실제 발화에 가까울수록:
- 다음 턴 응답의 맥락 일관성 향상
- 질문/감정/요청의 연속성 유지
- 페르소나 유지 + 내용 반응을 동시에 달성하기 쉬움

## 이번 1차 개선

- 오디오가 있는 요청에서는 audio_only 전사 프롬프트를 재사용해
  사용자 발화에 가까운 텍스트를 우선 추출
- addTurn(userText)에 baseText 대신 추출된 발화 텍스트를 저장
- 전사 추출 실패 시에만 기존 baseText를 fallback으로 사용

## 다음 턴 품질 차이 (요약)

- 개선 전: "직전 사용자"가 지시문이라 맥락 밀도가 낮음
- 개선 후: "직전 사용자"가 실제 발화에 가까워 맥락 밀도가 높음
- 기대 효과: 멀티턴에서 답변 연결감/자연스러움 상승
