# FINAL_DEMO_SCENARIOS

그림톡 최종 데모는 **mock으로 리허설 → live로 핵심만 검증** 순서로 진행한다.

- 모드 전환: `.env.local`의 `NEXT_PUBLIC_KANANA_MODE=mock | live`
- 서비스 흐름: 랜딩 → 카메라 → 미리보기 → 녹음 → 로딩 → 결과

---

## 시나리오 1) 이름 묻기형 (가장 기본)

- **목적**
  - “그림 친구가 살아난다”는 핵심 컨셉을 가장 안정적으로 전달

- **준비할 그림 예시**
  - 토끼 / 숲 / 귀가 강조된 귀여운 캐릭터

- **아이가 말할 추천 문장**
  - “안녕! 너 이름이 뭐야?”

- **기대되는 친구 응답 방향**
  - 자기소개 톤
  - 이름 관련 반응
  - 한 번 되묻는 질문(예: “넌 어떤 이름이 좋아?”)

- **성공 기준**
  - 결과 카드의 이름/정체/질문이 자연스럽다
  - assistantText가 발화와 연결된다
  - 오디오가 있으면 재생된다(없어도 UI가 어색하지 않음)

- **실패 시 fallback / 대체 시나리오**
  - 멘트: “그럼 이번엔 더 짧게 인사해볼게!”
  - 대체 발화: “안녕! 내 이름은 수영이야.”

- **mock 모드 사전 점검**
  - 카메라→미리보기→녹음→결과 흐름이 끊김 없이 보이는지
  - 결과 카드 위계(이름/질문/실제 응답)가 읽기 쉬운지

- **live 모드 검증 포인트**
  - 실제 응답이 “이름 묻기 의도”를 반영하는지
  - 대답 톤이 과도하게 기계적이지 않은지

---

## 시나리오 2) 같이 놀자형 (에너지 전달)

- **목적**
  - 상호작용성과 재미(놀이 제안)를 강하게 보여주기

- **준비할 그림 예시**
  - 공룡 / 뛰는 동물 / 움직임이 느껴지는 캐릭터

- **아이가 말할 추천 문장**
  - “같이 놀자!”
  - 또는 “오늘 뭐 하고 싶어?”

- **기대되는 친구 응답 방향**
  - 활기 있는 톤
  - 놀이/모험 제안
  - 다음 행동을 유도하는 짧은 질문

- **성공 기준**
  - 응답 에너지가 그림 분위기와 맞는다
  - 아이가 바로 다음 말을 하고 싶어지는 질문이 포함된다

- **실패 시 fallback / 대체 시나리오**
  - 멘트: “이번엔 더 또렷하게 한 번 더 말해볼게!”
  - 대체 발화: “우리 달리기 놀이 할래?”

- **mock 모드 사전 점검**
  - 녹음 버튼 상태(대기/녹음 중) 전달이 명확한지
  - 방금 대화 카드에서 user/assistant 구분이 잘 되는지

- **live 모드 검증 포인트**
  - 발화 의도(놀이 제안)를 반영해 응답이 나오는지
  - 결과 오디오 재생이 자연스러운지

---

## 시나리오 3) 위로/감정형 (감정 공감)

- **목적**
  - 단순 재미를 넘어 “감정 반응”이 가능한 경험을 보여주기

- **준비할 그림 예시**
  - 여우 / 별빛 숲 / 밤 분위기 / 조용한 친구

- **아이가 말할 추천 문장**
  - “조금 무서워.”
  - 또는 “같이 있어줄래?”

- **기대되는 친구 응답 방향**
  - 안심시키는 톤
  - 다정한 위로
  - 부담 없는 질문으로 대화 이어가기

- **성공 기준**
  - 응답이 위로 맥락을 반영한다
  - 말투가 부드럽고 안전하다
  - 결과 화면 문장 가독성이 좋다

- **실패 시 fallback / 대체 시나리오**
  - 멘트: “좋아, 이번엔 친구에게 같이 걷자고 말해볼게.”
  - 대체 발화: “천천히 같이 걸어줄래?”

- **mock 모드 사전 점검**
  - 로딩 화면 감정 톤(기다림→만남)이 자연스럽게 이어지는지
  - 결과 카드의 첫 인사/질문/실제 응답이 따뜻하게 보이는지

- **live 모드 검증 포인트**
  - 감정 키워드(무서워/같이 있어줘)를 실제 응답이 반영하는지
  - 과장되거나 어색한 문장이 없는지

---

## 데모 진행 권장 순서 (실전)

1. 이름 묻기형
2. 같이 놀자형
3. 위로/감정형

각 시나리오는 먼저 mock으로 1회 리허설한 뒤, live는 핵심 1회만 진행한다.
