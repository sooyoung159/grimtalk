# KANANA_CONVERSATION_STEP_DRAFT

## 목표

현재 `result -> record -> loading -> result` 반복 흐름을 유지하면서,
사용자가 연속 대화를 자연스럽게 이어갈 수 있도록 최소 확장한다.

## 현재 상태(요약)

- 첫 턴 결과가 result 화면에 표시됨
- 다시 말하기 버튼으로 record 단계로 돌아가 재요청 가능
- 대화 히스토리/턴 컨텍스트는 제한적

## 최소 확장안 (구조 유지)

1. 턴 개념 추가(경량)
- 각 요청을 `turn`으로 보고 번호만 증가
- 최근 N개(예: 3개) assistant/user 텍스트를 메모리에 보관

2. 다음 턴 요청 규칙
- image_audio 기본: 첫 턴 캡처 이미지를 재사용
- 사용자가 이미지 재촬영 원할 때만 camera로 이동
- 오디오는 턴마다 새로 녹음

3. 프롬프트 컨텍스트 최소 추가
- system은 고정
- user text에 직전 턴 요약 1~2줄만 포함
- 토큰 폭증 방지 위해 장기 히스토리는 미포함

4. UI 최소 변경
- result 화면 하단에 "방금 대화" 1~2개만 노출
- 기존 버튼 유지: 다시 말하기 / 처음부터

## 성공 기준

- 연속 3턴에서 캐릭터 페르소나 일관성 유지
- 직전 사용자 발화 반응률 개선
- 응답 길이 규칙(2~4문장, 마지막 질문) 유지

## 리스크

- 컨텍스트 과다 주입 시 답변 장문화
- 히스토리 누적 시 지연 증가
- 멀티모달+히스토리 조합에서 입력 우선순위 혼선

## 권장 구현 순서

1. turn 카운터 + 직전 1턴 텍스트만 주입
2. 품질 확인 후 직전 2~3턴으로 확장
3. 필요 시 conversation 전용 컴포넌트로 승격
