# RESULT_SCENE_V2_PLAN

## 왜 기존 결과 화면이 정보 화면처럼 느껴졌는가
- 이름, 정체, 성격, 응답이 같은 무게로 나열되어 카드식 정보 소비에 가까웠다.
- 사용자는 "만남"보다 "정리된 결과"를 읽는 느낌을 먼저 받았다.
- 설명 문구와 micro cue가 쌓이면서 감정보다 해설이 앞에 섰다.

## 왜 이번엔 첫 만남 장면 중심으로 바꾸는가
- grimtalk의 핵심은 캐릭터 정보 조회가 아니라, 내 그림 친구가 처음 살아나는 순간이다.
- 결과 화면 첫 인상은 정보 이해보다 감정 몰입을 우선해야 한다.
- 사용자가 "내가 만든 친구를 만났다"고 느껴야 다음 대화 행동도 자연스럽게 이어진다.

## 무엇을 앞으로 내리고 무엇을 뒤로 뺐는가
- 앞으로: 그림, 첫 인사 말풍선, 목소리 듣기, 다시 말 걸기.
- 뒤로: 정체/성격 같은 프로필성 정보, 실제 응답 원문, 방금 대화 기록.
- 즉시 보여야 하는 것은 장면 요소이고, 설명성 정보는 보조 영역으로 내렸다.

## 라벨/언어를 어떻게 바꿨는가
- "짜잔! 그림 친구를 만났어"보다 더 직접적으로 첫 만남을 느끼게 하는 문장으로 조정했다.
- "친구의 질문" → "친구가 이어서 물었어"처럼 장면 중심 언어로 바꿨다.
- 버튼/안내 문구도 기능 설명보다 감정 흐름을 해치지 않는 쪽으로 정리했다.

## 아이 입력의 흔적을 어떻게 강화했는가
- 사용자가 그린 그림을 결과 상단 핵심 시각으로 유지했다.
- 말풍선과 목소리 블록을 통해 "내 그림이 실제로 말한다"는 감각을 앞세웠다.
- 최근 대화에서도 "내가 한 말 / 친구가 한 말" 구분을 살려, 상호작용의 흔적이 남도록 했다.

## 이번 변경 후 기대 효과
- 결과 화면이 예쁜 요약 카드가 아니라 첫 만남 장면처럼 느껴진다.
- 사용자는 읽기보다 먼저 보고, 듣고, 다시 말 걸고 싶어진다.
- zero-text에 가까운 상태로 줄여도 핵심 감정 축(그림/말풍선/목소리/다시 말 걸기)이 유지된다.
